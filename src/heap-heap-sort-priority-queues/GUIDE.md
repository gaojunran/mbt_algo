# Heap / Heap Sort / Priority Queue — 手写指南

> 语言：MoonBit  
> 目标：从零实现一个二叉堆（Binary Heap），在此基础上实现堆排序和优先级队列，掌握数组模拟完全二叉树的核心思路。

---

## 一、核心概念

### 什么是堆

- **堆（Heap）** 是一棵**完全二叉树**，满足**堆性质（heap property）**：
  - **最大堆（Max-Heap）**：每个节点的值 ≥ 其所有子节点的值 → 根节点是最大值。
  - **最小堆（Min-Heap）**：每个节点的值 ≤ 其所有子节点的值 → 根节点是最小值。
- 堆**不是** BST——左右子节点之间没有大小关系，只有父子之间有约束。

### 为什么用数组

- 完全二叉树可以用**数组**紧凑地表示，不需要指针/引用。
- 对于下标从 **0** 开始的数组：
  - 节点 `i` 的**左子节点**：`2 * i + 1`
  - 节点 `i` 的**右子节点**：`2 * i + 2`
  - 节点 `i` 的**父节点**：`(i - 1) / 2`（整数除法）
- 这三个公式是堆操作的基础，务必牢记。

### 堆、堆排序、优先级队列的关系

```
二叉堆（数据结构）
  ├── 堆排序（排序算法）：利用堆的性质进行原地排序
  └── 优先级队列（抽象数据类型）：用堆作为底层实现，支持高效的插入和取最值
```

## 二、数据结构定义

### 需要实现什么

- **堆（Heap）**：一个包装类型，内部持有一个动态数组（`Array[T]`）和当前大小（`size`）。
- 键的类型需要支持**比较操作**（`Compare` trait）。

### 用什么实现

- 使用 MoonBit 的 **`struct`** 来定义堆，内部用 `Array[T]` 存储元素。
- 你需要决定实现**最大堆**还是**最小堆**——建议先实现**最大堆**，因为堆排序天然需要最大堆来产生升序结果。

### 注意事项

- 数组下标从 0 开始，所有父子关系的公式都基于此。
- `size` 字段记录堆中有效元素的个数，它可能小于数组的实际容量。
- 先想清楚你的堆是泛型的（`Heap[T : Compare]`）还是只支持 `Int`——建议先用 `Int` 练习，理解后再泛化。

## 三、核心操作——二叉堆（最大堆）

按照优先级，你需要依次实现以下操作：

### 1. 辅助函数：swap 和索引计算

在开始核心操作之前，先准备好这些工具函数：

- `swap(i, j)` → 交换数组中下标 `i` 和 `j` 的元素。
- `parent(i)` → 返回 `(i - 1) / 2`。
- `left_child(i)` → 返回 `2 * i + 1`。
- `right_child(i)` → 返回 `2 * i + 2`。

### 2. 上浮（sift_up / bubble_up / percolate_up）

这是**插入操作**的核心子过程。

- 场景：一个新元素被放在数组末尾，可能违反堆性质（比父节点大）。
- 做法：不断将该元素与其父节点比较，如果比父节点大就交换，直到满足堆性质或到达根节点。

```
sift_up(i):
│
├─ i == 0 → 已到根节点，停止
│
└─ arr[i] > arr[parent(i)] → swap(i, parent(i))
                               继续 sift_up(parent(i))
   arr[i] <= arr[parent(i)] → 停止，堆性质已满足
```

- 时间复杂度：O(log n)，最多沿着树的高度走一遍。

### 3. 下沉（sift_down / bubble_down / percolate_down）

这是**删除操作**和**建堆操作**的核心子过程。

- 场景：根节点（或某个节点）的值可能比子节点小，违反堆性质。
- 做法：将该节点与其**较大的子节点**比较，如果比子节点小就交换，然后继续向下，直到满足堆性质或到达叶子节点。

```
sift_down(i):
│
├─ 计算 left = 2*i+1, right = 2*i+2
├─ 找出 i, left, right 三者中值最大的，记为 largest
│
├─ largest == i → 堆性质已满足，停止
└─ largest != i → swap(i, largest)
                   继续 sift_down(largest)
```

#### ⚠️ 关键细节

- 比较时要先检查 `left` 和 `right` 是否越界（`< size`）。
- 必须选**较大的**子节点交换，否则交换后可能仍然违反堆性质。
- 时间复杂度：O(log n)。

### 4. 插入（insert / push）

```
insert(val):
  1. 将 val 追加到数组末尾
  2. size += 1
  3. sift_up(size - 1)  // 对新元素执行上浮
```

- 时间复杂度：O(log n)。

### 5. 取最大值（peek / top）

- 直接返回 `arr[0]`（根节点）。
- 注意处理空堆的情况（返回 `None` 或报错）。
- 时间复杂度：O(1)。

### 6. 删除最大值（extract_max / pop）

这是堆操作中**最容易出错**的部分。

```
extract_max():
  1. 记录 arr[0] 作为返回值
  2. 将数组最后一个元素移到 arr[0]
  3. size -= 1
  4. sift_down(0)  // 对根节点执行下沉
  5. 返回记录的最大值
```

#### 为什么不直接删除根节点？

- 如果直接移除 `arr[0]`，数组中间会出现空洞，完全二叉树的结构就被破坏了。
- 把最后一个元素放到根的位置，保证了结构完整性，然后用 `sift_down` 恢复堆性质。

- 时间复杂度：O(log n)。

### 7. 建堆（build_heap / heapify）

从一个无序数组构建堆，有两种方式：

#### 方式 A：逐个插入（自顶向下）

- 依次对每个元素调用 `insert`。
- 时间复杂度：O(n log n)。

#### 方式 B：原地建堆（自底向上）⭐ 推荐

- 从**最后一个非叶子节点**开始，逆序对每个节点执行 `sift_down`。
- 最后一个非叶子节点的下标：`size / 2 - 1`。

```
build_heap(arr):
  for i = size/2 - 1 downto 0:
    sift_down(i)
```

- 时间复杂度：**O(n)**（不是 O(n log n)！这是一个经典的数学证明，底层节点多但下沉距离短）。
- 这个方法更优，堆排序中也使用这种方式。

## 四、堆排序（Heap Sort）

### 思路

堆排序利用最大堆的性质：根节点总是最大值。反复取出最大值放到数组末尾，就能得到升序序列。

### 算法步骤

```
heap_sort(arr):
  1. build_heap(arr)           // O(n) 原地建最大堆
  2. for i = n-1 downto 1:
       swap(arr[0], arr[i])    // 把当前最大值放到末尾
       size -= 1               // 缩小堆的范围
       sift_down(0)            // 恢复堆性质
```

### 流程图

```
原始数组: [4, 1, 3, 9, 7]

Step 1: build_heap → [9, 7, 3, 1, 4]  (最大堆)

Step 2: 反复 extract_max
  swap(9,4) → [4, 7, 3, 1, | 9]   sift_down → [7, 4, 3, 1, | 9]
  swap(7,1) → [1, 4, 3, | 7, 9]   sift_down → [4, 1, 3, | 7, 9]
  swap(4,3) → [3, 1, | 4, 7, 9]   sift_down → [3, 1, | 4, 7, 9]
  swap(3,1) → [1, | 3, 4, 7, 9]   sift_down → [1, | 3, 4, 7, 9]

结果: [1, 3, 4, 7, 9] ✅
```

### 注意事项

- 堆排序是**原地排序**（in-place），只需要 O(1) 额外空间。
- 堆排序是**不稳定排序**（相等元素的相对顺序可能改变）。
- 时间复杂度：O(n log n)，无论最好、最坏、平均都是 O(n log n)。
- `|` 右边的部分是已排好序的区域，不再属于堆。

## 五、优先级队列（Priority Queue）

### 什么是优先级队列

- 优先级队列是一种**抽象数据类型（ADT）**，支持：
  - `push(val)` — 插入一个元素
  - `pop()` — 取出并删除优先级最高的元素
  - `peek()` — 查看优先级最高的元素（不删除）
- 用**最大堆**实现 → 最大优先级队列（优先级最高 = 值最大）。
- 用**最小堆**实现 → 最小优先级队列（优先级最高 = 值最小）。

### 实现

如果你已经实现了二叉堆，优先级队列就是对堆操作的**薄封装**：

| 优先级队列操作 | 对应堆操作 |
|---|---|
| `push(val)` | `insert(val)` |
| `pop()` | `extract_max()` / `extract_min()` |
| `peek()` | 返回 `arr[0]` |
| `is_empty()` | `size == 0` |
| `size()` | 返回 `size` |

### 注意事项

- 优先级队列的核心价值在于 `push` 和 `pop` 都是 O(log n)，而 `peek` 是 O(1)。
- 如果用无序数组实现，`push` 是 O(1) 但 `pop` 是 O(n)；如果用有序数组实现，`pop` 是 O(1) 但 `push` 是 O(n)。堆是两者的最佳折中。

## 六、实现顺序建议

```
1. 定义 struct 类型（数组 + size）
2. 辅助函数：parent / left_child / right_child / swap
3. sift_up
4. insert（调用 sift_up）
5. peek
6. sift_down
7. extract_max（调用 sift_down）
8. 用 insert + extract_max 验证基本功能
9. build_heap（自底向上，调用 sift_down）
10. heap_sort（调用 build_heap + sift_down）
11. 封装为 PriorityQueue（可选）
```

## 七、关键注意事项

### 关于堆性质

- **核心不变量**：对于任意节点 `i`，`arr[i] >= arr[left_child(i)]` 且 `arr[i] >= arr[right_child(i)]`（最大堆）。
- 堆只保证父子关系，**不保证**左右子节点的大小关系，也**不保证**同层节点的顺序。
- 堆**不是**有序数组——除了根节点是最值之外，其他元素的位置没有全局意义。

### 关于 sift_down 的易错点

- **必须和较大的子节点交换**（最大堆），不能随便选一个。否则交换后父节点可能仍然小于另一个子节点。
- **必须检查子节点下标是否越界**。一个常见 bug 是忘记检查右子节点是否存在（完全二叉树的最后一层可能只有左子节点没有右子节点）。
- 循环/递归的终止条件：当前节点已经是三者中最大的，或者已经没有子节点。

### 关于建堆

- 自底向上建堆的起点是 `size / 2 - 1`，不是 `size - 1`。叶子节点不需要 `sift_down`，因为它们没有子节点，天然满足堆性质。
- 叶子节点占了数组的大约一半，跳过它们是 O(n) 建堆的关键。

### 关于堆排序

- 堆排序的第二阶段（反复 extract_max）中，`sift_down` 的范围在不断缩小——每次 `size` 减 1。如果你的 `sift_down` 使用了全局的 `size` 而不是传入的参数，会导致 bug。
- 建议 `sift_down` 接受一个 `heap_size` 参数，而不是直接读取堆的 `size` 字段。

### 关于性能

| 操作 | 时间复杂度 |
|---|---|
| `insert` | O(log n) |
| `peek` | O(1) |
| `extract_max` | O(log n) |
| `build_heap` | O(n) |
| `heap_sort` | O(n log n) |

- 堆排序的常数因子比快速排序大（缓存不友好），实际中快速排序通常更快。
- 但堆排序的最坏情况是 O(n log n)，而快速排序最坏是 O(n²)。

### 关于测试

- 每次 `insert` 后，验证堆性质：对所有 `i`，检查 `arr[i] >= arr[left]` 且 `arr[i] >= arr[right]`。
- 反复 `extract_max` 应该得到一个**降序**序列。
- 堆排序后数组应该是**升序**的。
- 测试边界情况：空堆 pop、只有一个元素的堆、所有元素相同。

## 八、测试建议

### 测试用例 1：基本插入和取最大值

依次插入 `[4, 1, 3, 9, 7, 5, 2]`，每次插入后验证 `peek()` 返回当前最大值：

```
insert(4) → peek = 4
insert(1) → peek = 4
insert(3) → peek = 4
insert(9) → peek = 9
insert(7) → peek = 9
insert(5) → peek = 9
insert(2) → peek = 9
```

### 测试用例 2：反复 extract_max

在上面的堆上反复调用 `extract_max()`，应该得到：`9, 7, 5, 4, 3, 2, 1`。

### 测试用例 3：堆排序

```
输入: [3, 1, 4, 1, 5, 9, 2, 6, 5, 3]
输出: [1, 1, 2, 3, 3, 4, 5, 5, 6, 9]
```

### 测试用例 4：建堆验证

对数组 `[1, 2, 3, 4, 5, 6, 7]` 执行 `build_heap`，结果应该是一个合法的最大堆（根节点为 7），可以通过检查每个父节点 ≥ 子节点来验证。

## 九、延伸思考

完成基础堆和堆排序后，可以思考以下问题：

1. 如何实现**最小堆**？只需要改变比较方向，还是有更优雅的方式（比如传入比较函数）？
2. 如何支持 **decrease_key / increase_key** 操作？（这在 Dijkstra 算法中很重要）
3. **d 叉堆（d-ary heap）**：如果每个节点有 d 个子节点而不是 2 个，性能会如何变化？
4. 了解**斐波那契堆（Fibonacci Heap）**——它的 decrease_key 是 O(1) 摊还的，为什么这很重要？
5. **Top-K 问题**：如何用堆高效地找到 n 个元素中最大的 k 个？（提示：用大小为 k 的最小堆）
