# Red-Black Tree — 手写指南

> 语言：MoonBit  
> 目标：从零手写红黑树（Red-Black Tree），掌握“局部旋转 + 重新着色”维持平衡的核心机制，并能独立实现插入与删除。

---

## 一、核心概念

### 什么是红黑树

- 红黑树是一种自平衡二叉搜索树（BST）。
- 它通过给节点增加“颜色位”（红或黑），在每次插入/删除后做有限次修复，保证树高度保持在 O(log n)。
- 红黑树不是严格平衡（不像 AVL 那么“紧”），但旋转次数更少，工程上很常用。

### 五条性质（必须始终成立）

1. 每个节点要么是红色，要么是黑色。
2. 根节点是黑色。
3. 所有空叶子（NIL，哨兵）是黑色。
4. 红色节点的两个子节点都必须是黑色（不能出现连续红）。
5. 从任一节点到其所有后代 NIL 叶子的路径上，黑色节点数量相同（黑高一致）。

### 为什么能保持平衡

- 性质 4 限制了红节点不能连续出现。
- 性质 5 保证任一路径黑节点数一致。
- 这两者共同约束了最长路径不会超过最短路径的 2 倍，因此高度是 O(log n)。

## 二、问题建模

### 输入和输出

- 输入：一系列操作（插入 key、删除 key、查找 key）。
- 输出：满足红黑树性质的 BST 结构。

### 节点结构设计

建议节点至少包含：

- `key`：键（可比较）
- `value`：（可选）值
- `color`：`Red | Black`
- `left` / `right`：左右孩子
- `parent`：父节点（写修复逻辑非常方便）

### NIL 哨兵建议

- 手写时强烈建议使用统一的黑色 NIL 哨兵，而不是 `None`。
- 这样旋转、删除修复时边界判断会明显更清晰。

---

## 三、基础操作：旋转

红黑树平衡修复的本质是两种局部变换。

### 左旋（left rotate）

对节点 `x` 左旋，等价于让 `x.right`（记为 `y`）上升，`x` 下沉为 `y.left`。

```
		x                 y
		 \               / \
			y     =>      x   c
		 / \             \
		b   c             b
```

### 右旋（right rotate）

与左旋对称。

```
			y               x
		 /               / \
		x      =>       a   y
	 / \                 /
	a   b               b
```

旋转不会破坏 BST 中序有序性，只改变局部高度与父子关系。

---

## 四、插入流程（Insert）

### 第一步：按 BST 规则插入

- 新节点先按普通 BST 插入到叶子位置。
- 新节点初始颜色设为红色。

为什么先设红色：

- 若设黑色，会直接影响黑高，修复更复杂。
- 设红色通常只会触发“连续红”问题，修复局部即可。

### 第二步：插入修复（fix_insert）

只要父节点是红色，就违反性质 4，需要循环修复。

设：

- `z`：当前节点
- `p`：`z.parent`
- `g`：`p.parent`
- `u`：`z` 的叔叔节点（`g` 的另一个孩子）

分三类（含镜像对称）：

1. 叔叔是红色（Case 1）
- `p` 和 `u` 染黑，`g` 染红。
- `z = g`，继续向上修。

2. 叔叔是黑色，且 `z` 与 `p` 形成“内拐”（Case 2）
- 例如 `p` 是左孩子而 `z` 是右孩子。
- 先对 `p` 旋转一次，把结构转成“外拐”。

3. 叔叔是黑色，且 `z` 与 `p` 形成“外拐”（Case 3）
- 例如 `p` 是左孩子且 `z` 是左孩子。
- 对 `g` 旋转一次，并交换 `p`/`g` 颜色。

循环结束后，强制根节点为黑。

---

## 五、删除流程（Delete）

删除是红黑树里最难的部分。

### 第一步：按 BST 规则删除

- 若目标节点 `z` 至多一个非 NIL 孩子，直接删除。
- 若有两个孩子，用后继节点 `y` 替换 `z`（经典 BST 删除套路），实际删除 `y`。

关键点：记录“被真正移除节点的原颜色”。

- 若删掉的是红节点，通常无需进一步修复。
- 若删掉的是黑节点，会造成黑高缺失，需要 `fix_delete`。

### 第二步：删除修复（fix_delete）

把“多出来的一重黑”看作挂在节点 `x` 上，循环处理直到平衡恢复。

设兄弟节点为 `w`，分四类（同样有镜像）：

1. `w` 是红色
- 把 `w` 染黑、父节点染红，对父节点旋转。
- 作用是把问题转换成“兄弟为黑”的情形。

2. `w` 是黑色，且 `w` 的两个孩子都黑
- `w` 染红，`x` 上移到父节点，继续循环。

3. `w` 是黑色，`w` 的近侄子红、远侄子黑
- 先对 `w` 旋转并重新着色，转成 Case 4。

4. `w` 是黑色，`w` 的远侄子红
- 用父节点颜色覆盖 `w`，父节点染黑，远侄子染黑。
- 对父节点旋转，修复完成并退出循环。

最后把 `x` 染黑。

## 六、易错点与调试建议

### 1. 忘记维护 parent 指针

- 旋转后必须同步更新多处 `parent`。
- parent 一旦错，后续修复会出现“看起来像随机”的错误。

### 2. NIL 处理不统一

- 一会儿用 `None`，一会儿用哨兵，逻辑会非常混乱。
- 建议全程统一黑色 NIL。

### 3. 插入修复漏掉镜像分支

- 只写了“父在祖父左侧”的三类，忘记“父在祖父右侧”的对称逻辑。

### 4. 删除修复 case 顺序写反

- 删除修复很依赖 case 递进关系。
- 建议严格按 Case 1 -> 2 -> 3 -> 4 编码，并在注释标注。

### 5. 根颜色未回收为黑

- 插入修复结束后，必须显式 `root.color = Black`。

---

## 七、复杂度分析

设节点数为 `n`：

- 查找：O(log n)
- 插入：O(log n)
- 删除：O(log n)
- 空间：O(n)

红黑树通过局部旋转和有限次着色，把最坏高度控制在 O(log n)。

---

## 八、实现顺序建议（MoonBit）

```
1. 定义 Color、Node、NIL 哨兵
2. 实现基础 BST 查找与插入定位
3. 实现 left_rotate / right_rotate（先单测旋转）
4. 实现 insert + fix_insert
5. 实现 transplant（删除时替换子树的工具函数）
6. 实现 delete + fix_delete
7. 实现 inorder 验证 BST 有序性
8. 实现 verify_rb_properties（自动校验 5 条性质）
9. 增加随机测试：随机插删后持续校验
```

---

## 九、测试建议

至少覆盖这些测试：

- 基础插入：插入递增序列后，树高度仍在对数级。
- 基础删除：删除叶子、删除单孩子节点、删除双孩子节点。
- 重复随机操作：随机插入/删除后，始终满足红黑性质。
- 有序性：中序遍历结果始终严格递增。
- 边界：空树、单节点树、删除根节点。

实用断言方向：

```
1. is_bst(root) == true
2. root.color == Black
3. no_red_red_parent_child(root) == true
4. black_height_consistent(root) == true
```

只要你把“旋转正确 + 修复 case 完整 + 性质校验函数”三件事做好，红黑树实现就会很稳。
