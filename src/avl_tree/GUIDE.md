# AVL Tree — 手写指南

> 语言：MoonBit  
> 目标：从零手写 AVL 树，掌握“平衡因子 + 旋转修复”的完整逻辑，并能稳定实现插入、删除与校验。

---

## 一、核心概念

### 什么是 AVL 树

- AVL 树是最经典的自平衡二叉搜索树（BST）。
- 对任意节点，左子树和右子树高度差最多为 1。
- 因为平衡约束更严格，查询性能非常稳定，最坏高度是 O(log n)。

### 平衡因子（Balance Factor）

- 定义：`bf(node) = height(left) - height(right)`
- AVL 要求：每个节点都满足 `bf ∈ {-1, 0, 1}`

当某个节点 `|bf| > 1`，就失衡，需要旋转修复。

### 为什么能保持高效

- 插入/删除后只会影响从修改点到根的一条路径。
- 每个失衡节点最多通过常数次旋转恢复平衡。
- 所以整体插入/删除仍是 O(log n)。

---

## 二、问题建模

### 输入和输出

- 输入：操作序列（插入 key、删除 key、查找 key）。
- 输出：始终保持 AVL 平衡约束的 BST 结构。

### 节点结构设计

建议节点至少包含：

- `key`：可比较键
- `value`：（可选）值
- `left` / `right`：左右子树
- `height`：该节点高度（强烈建议缓存）

可选：

- `parent`：父指针（便于向上回溯修复）

### 高度约定

建议固定一套约定并全程一致：

- 空节点高度 = `0`
- 叶子节点高度 = `1`

这样更新公式统一为：

`height(node) = 1 + max(height(left), height(right))`

---

## 三、基础操作：旋转

AVL 只有两种基础旋转，另外两类是组合旋转。

### 右旋（Right Rotate）

对 `y` 右旋，`x = y.left` 上升：

```
	  y                x
	 /                / \
	x      =>        a   y
   / \                  / \
  a   b                b   c
	   \
		c
```

### 左旋（Left Rotate）

对 `x` 左旋，`y = x.right` 上升：

```
	  x                    y
   / \                  / \
  a   y      =>        x   c
		/ \              / \
		b   c            a   b
```

### 旋转后必须做的事

- 正确重连子树指针。
- 按“先低后高”顺序更新相关节点 `height`。

---

## 四、四种失衡类型与修复

设失衡节点为 `z`。

### 1. LL 型（左左）

- 条件：`bf(z) > 1` 且 `bf(z.left) >= 0`
- 修复：对 `z` 做一次右旋。

### 2. RR 型（右右）

- 条件：`bf(z) < -1` 且 `bf(z.right) <= 0`
- 修复：对 `z` 做一次左旋。

### 3. LR 型（左右）

- 条件：`bf(z) > 1` 且 `bf(z.left) < 0`
- 修复：先对 `z.left` 左旋，再对 `z` 右旋。

### 4. RL 型（右左）

- 条件：`bf(z) < -1` 且 `bf(z.right) > 0`
- 修复：先对 `z.right` 右旋，再对 `z` 左旋。

记忆口诀：

- “重在哪边先转子，最后转根”。

---

## 五、插入流程（Insert）

### 第一步：按 BST 规则插入

- 像普通 BST 一样找到位置插入新节点。

### 第二步：向上回溯更新并修复

- 从插入点父节点开始一路向上：
1. 更新当前节点高度。
2. 计算平衡因子。
3. 若失衡，判断 LL/RR/LR/RL 并旋转。

关键点：

- 插入时通常在第一个失衡祖先修复后，整棵树就恢复平衡（因为该子树高度回到插入前水平或可控变化）。

---

## 六、删除流程（Delete）

删除比插入更容易连锁失衡。

### 第一步：按 BST 规则删除

- 0 或 1 个孩子：直接删。
- 2 个孩子：用后继（或前驱）替换，再删除后继（前驱）节点。

### 第二步：向上回溯更新并修复

- 从“实际被删位置的父节点”向上处理：
1. 更新高度。
2. 计算平衡因子。
3. 失衡则按四类旋转修复。

关键差异：

- 删除后可能一路向上都需要修复，不要像插入一样“修一次就停”的思维。

---

## 七、易错点与调试建议

### 1. 高度更新顺序错误

- 旋转后如果先更新新根再更新旧根，可能读到过期高度。
- 建议每个旋转函数内固定顺序并封装好。

### 2. 平衡因子符号写反

- 一定统一 `bf = left - right`。
- 若改成 `right - left`，四类判断条件要全部镜像修改。

### 3. 删除后漏掉持续回溯

- 删除可能让多个祖先节点依次失衡。
- 必须沿父链继续处理到根。

### 4. 旋转后父子链接没接完整

- 尤其在有 `parent` 指针时，常漏改 1~2 处引用。
- 建议单独给旋转写小测试，先确认结构正确再接入插删。

### 5. 重复 key 策略不明确

- 提前定义：忽略、覆盖值、还是计数。
- 测试需覆盖该策略。

---

## 八、复杂度分析

设节点数为 `n`：

- 查找：O(log n)
- 插入：O(log n)
- 删除：O(log n)
- 空间：O(n)

AVL 平衡更严格，查询路径通常比红黑树更短，但更新时旋转可能稍多。

## 九、实现顺序建议（MoonBit）

```
1. 定义 Node（含 height）与基础工具函数：height/get_bf/update_height
2. 实现 left_rotate / right_rotate（先做单元测试）
3. 实现 BST insert 骨架
4. 在 insert 回溯中接入四类修复
5. 实现 BST delete 骨架（含后继替换）
6. 在 delete 回溯中接入四类修复
7. 实现 inorder（验证有序性）
8. 实现 verify_avl（验证高度与平衡因子）
9. 加随机插删回归测试
```

## 十、测试建议

至少覆盖以下测试：

- 基本插入：插入升序/降序序列后仍保持平衡。
- 四种旋转：人为构造 LL、RR、LR、RL 样例。
- 基本删除：删叶子、删单孩子、删双孩子。
- 连续删除：多次删除后仍满足 AVL 约束。
- 随机测试：随机插删若干轮后校验结构。
- 边界：空树、单节点、重复 key。

实用断言方向：

```
1. is_bst(root) == true
2. abs(balance_factor(node)) <= 1 for all nodes
3. node.height == 1 + max(height(node.left), height(node.right)) for all nodes
```

如果你把“高度维护正确 + 四类旋转判断正确 + 删除持续回溯”这三件事写稳，AVL 就基本不会出大问题。
