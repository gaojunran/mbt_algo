# Binary Search Tree — 手写指南

> 语言：MoonBit  
> 目标：从零实现一棵泛型二叉搜索树，掌握递归数据结构与递归/迭代算法的核心思路。

---

## 一、数据结构定义

### 需要实现什么

- **节点（Node）**：每个节点存储一个键（key）、一个值（value，可选）、左子树指针、右子树指针。
- **树（BST）**：持有根节点的引用，可以是一个包装类型，也可以直接用节点的枚举类型表示。

### 用什么实现

- 使用 MoonBit 的 **`enum`（代数数据类型）** 来表示树的递归结构，例如 `Empty | Node(...)` 两个变体。
- 键的类型需要支持**比较操作**（`Compare` trait），这样才能决定左/右子树的归属。

### 注意事项

- 想清楚你的树是只存 key，还是存 key-value 对——这会影响后续所有操作的签名。
- MoonBit 中 `enum` 是不可变的值类型，如果你想要可变版本，需要用 `struct` + `Option` 来建模，或者采用纯函数式风格（每次操作返回新树）。
- 推荐先用**纯函数式（immutable）**风格练习，理解起来更简单，之后再尝试可变版本。

---

## 二、核心操作

按照优先级，你需要依次实现以下操作：

### 1. 插入（insert）

- 递归地比较 key，决定往左子树还是右子树走，直到找到 `Empty` 位置，创建新节点。
- 思考：如果插入的 key 已经存在，是覆盖值、忽略、还是报错？

### 2. 查找（search / lookup / contains）

- 递归地比较 key，沿着左/右子树查找，找到返回对应值，找不到返回 `None`。
- 这是 BST 最基本的操作，也是理解 BST 性质的关键。

### 3. 删除（remove / delete）

- 这是 BST 中**最复杂**的操作，需要分三种情况处理：
  1. 待删除节点是**叶子节点**（无子节点）
  2. 待删除节点有**一个子节点**
  3. 待删除节点有**两个子节点**（需要找到中序前驱或中序后继来替换）
- 建议最后再实现这个操作。

### 4. 遍历（traversal）

- **中序遍历（in-order）**：左 → 根 → 右，结果是有序序列——这是验证 BST 正确性的最佳手段。
- **前序遍历（pre-order）**：根 → 左 → 右。
- **后序遍历（post-order）**：左 → 右 → 根。
- 可以返回一个 `Array`，也可以接受一个回调函数（visitor 模式）。

---

## 三、辅助操作（可选但推荐）

| 操作 | 说明 |
|---|---|
| `min` / `max` | 找到树中最小/最大的 key（一路向左/向右） |
| `size` | 返回节点总数 |
| `height` / `depth` | 返回树的高度 |
| `is_empty` | 判断树是否为空 |
| `from_array` | 从数组批量构建 BST |
| `to_sorted_array` | 中序遍历导出有序数组 |
| `Show` trait 实现 | 方便调试时打印树的结构 |

---

## 四、实现顺序建议

```
1. 定义 enum 类型
2. insert
3. search
4. in-order traversal（用来验证 insert 是否正确）
5. min / max
6. delete（先处理简单情况，再处理两个子节点的情况）
7. 其他辅助操作
```

## 五、关键注意事项

### 关于 BST 性质

- **核心不变量**：对于任意节点，左子树所有 key < 当前 key < 右子树所有 key。
- 注意是**所有**，不仅仅是直接子节点。删除操作中如果破坏了这个性质，整棵树就废了。

### 关于递归

- BST 的几乎所有操作都天然适合递归实现。
- 纯函数式风格下，每次操作都返回一棵"新树"（实际上大部分节点是共享的）。
- 注意递归的**基准情况（base case）**：通常是 `Empty`。

### 关于性能

- BST 的操作时间复杂度取决于树的高度：平均 O(log n)，最坏 O(n)。
- 最坏情况发生在**有序插入**时，树退化为链表。
- 如果你发现自己的 BST 在有序数据上表现很差，这是正常的——这正是后续学习 AVL 树、红黑树的动机。

### 关于测试

- 用中序遍历验证 BST 性质是最简单有效的方法。
- 测试删除操作时，务必覆盖三种情况（叶子、单子节点、双子节点）。
- 测试边界情况：空树操作、只有一个节点的树、重复 key 插入。

## 六、延伸思考

完成基础 BST 后，可以思考以下问题：

1. 如何让 BST 自平衡？（→ AVL 树、红黑树）
2. 如何实现迭代版本的遍历？（用显式栈模拟递归）
3. 如何支持范围查询（range query）？
4. BST 和排序算法（tree sort）之间的关系是什么？
