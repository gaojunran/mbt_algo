# Binary Search Tree — 手写指南

> 语言：MoonBit  
> 目标：从零实现一棵泛型二叉搜索树，掌握递归数据结构与递归/迭代算法的核心思路。

---

## 一、数据结构定义

### 需要实现什么

- **节点（Node）**：每个节点存储一个键（key）、一个值（value，可选）、左子树指针、右子树指针。
- **树（BST）**：持有根节点的引用，可以是一个包装类型，也可以直接用节点的枚举类型表示。

### 用什么实现

- 使用 MoonBit 的 **`enum`（代数数据类型）** 来表示树的递归结构，例如 `Empty | Node(...)` 两个变体。
- 键的类型需要支持**比较操作**（`Compare` trait），这样才能决定左/右子树的归属。

### 注意事项

- 想清楚你的树是只存 key，还是存 key-value 对——这会影响后续所有操作的签名。
- MoonBit 中 `enum` 是不可变的值类型，如果你想要可变版本，需要用 `struct` + `Option` 来建模，或者采用纯函数式风格（每次操作返回新树）。
- 推荐先用**纯函数式（immutable）**风格练习，理解起来更简单，之后再尝试可变版本。

## 二、核心操作

按照优先级，你需要依次实现以下操作：

### 1. 插入（insert）

- 递归地比较 key，决定往左子树还是右子树走，直到找到 `Empty` 位置，创建新节点。
- 思考：如果插入的 key 已经存在，是覆盖值、忽略、还是报错？

### 2. 查找（search / lookup / contains）

- 递归地比较 key，沿着左/右子树查找，找到返回对应值，找不到返回 `None`。
- 这是 BST 最基本的操作，也是理解 BST 性质的关键。

### 3. 删除（remove / delete）

这是 BST 中**最复杂**的操作。整体分为两大步：**定位节点** 和 **处理删除**。

#### 第一步：定位待删除节点

和 `search` 一样，递归地比较 val，沿左/右子树向下查找：
- `val < node.val` → 递归进左子树，将返回值赋给 `node.left`
- `val > node.val` → 递归进右子树，将返回值赋给 `node.right`
- `val == node.val` → 找到了，进入第二步

> ⚠️ 如果一路走到 `None` 都没找到，说明值不存在，直接返回 `None`。

#### 第二步：处理删除（三种情况）

找到目标节点后，根据其子节点情况分别处理：

**情况 1：叶子节点（左右子树都为空）**
- 最简单——直接返回 `None`，相当于把这个节点从树中移除。

**情况 2：只有一个子节点**
- 左子树为空 → 返回右子树
- 右子树为空 → 返回左子树
- 本质上就是"跳过"当前节点，让子节点顶上来。

**情况 3：有两个子节点** ⭐ 最关键
- 你不能简单地删掉节点，因为它有两棵子树需要"安置"。
- 策略：找到一个**替身**来代替被删节点的位置，这个替身必须满足 BST 性质。
- 有两个合法的替身选择：
  - **中序后继（in-order successor）**：右子树中的最小值节点（右子树中一路向左）
  - **中序前驱（in-order predecessor）**：左子树中的最大值节点（左子树中一路向右）
- 选定替身后，操作步骤：
  1. 找到替身节点的值
  2. 用替身的值**覆盖**当前节点的值
  3. 在对应的子树中**递归删除**替身节点（因为替身最多只有一个子节点，所以会落入情况 1 或情况 2）

> 💡 推荐使用**中序后继**（右子树最小值），因为它一定没有左子节点，删除它时只需处理情况 1 或情况 2。

#### 你需要的辅助函数

- `find_min(node)` → 找到以 `node` 为根的子树中的最小值节点（一路向左直到 `left` 为 `None`）。
- 这个函数在删除的情况 3 中会用到。

#### 关于你当前代码的提示

你现在的 `delete_node` 有两个问题需要修正：
1. **缺少定位逻辑**：当前代码找到节点后直接处理删除，但没有递归地根据 `val` 的大小向左/右子树查找——你需要先比较 `val` 和 `node.val`，只有相等时才进入删除逻辑。
2. **缺少情况 3**：`match` 的分支没有覆盖 `(Some(_), Some(_))` 的情况——这正是需要用中序后继来处理的场景。

#### 删除操作的流程图

```
delete_node(root, val)
│
├─ root 为 None → 返回 None（值不存在）
│
├─ val < node.val → node.left = delete_node(node.left, val)，返回 root
├─ val > node.val → node.right = delete_node(node.right, val)，返回 root
│
└─ val == node.val（找到目标）
   │
   ├─ 左右都为 None → 返回 None
   ├─ 只有右子树   → 返回右子树
   ├─ 只有左子树   → 返回左子树
   └─ 左右都有     → 找右子树最小值 min_val
                      node.val = min_val
                      node.right = delete_node(node.right, min_val)
                      返回 root
```

```

       / \
```

然后分别测试：

- 删除单子节点的节点：先 `delete(4)` 让 3 变成只有左子节点，再 `delete(3)`

- 每次删除后用 `inorder()` 验证结果仍然有序

### 4. 遍历（traversal）

- **中序遍历（in-order）**：左 → 根 → 右，结果是有序序列——这是验证 BST 正确性的最佳手段。
- **前序遍历（pre-order）**：根 → 左 → 右。
- **后序遍历（post-order）**：左 → 右 → 根。
- 可以返回一个 `Array`，也可以接受一个回调函数（visitor 模式）。

## 三、辅助操作（可选但推荐）

| 操作 | 说明 |
|---|---|
| `min` / `max` | 找到树中最小/最大的 key（一路向左/向右） |
| `size` | 返回节点总数 |
| `height` / `depth` | 返回树的高度 |
| `is_empty` | 判断树是否为空 |
| `from_array` | 从数组批量构建 BST |
| `to_sorted_array` | 中序遍历导出有序数组 |
| `Show` trait 实现 | 方便调试时打印树的结构 |

```

2. insert
3. search

5. min / max

```

## 五、关键注意事项

### 关于 BST 性质

- **核心不变量**：对于任意节点，左子树所有 key < 当前 key < 右子树所有 key。
- 注意是**所有**，不仅仅是直接子节点。删除操作中如果破坏了这个性质，整棵树就废了。

### 关于递归

- BST 的几乎所有操作都天然适合递归实现。
- 纯函数式风格下，每次操作都返回一棵"新树"（实际上大部分节点是共享的）。
- 注意递归的**基准情况（base case）**：通常是 `Empty`。

### 关于性能

- BST 的操作时间复杂度取决于树的高度：平均 O(log n)，最坏 O(n)。
- 最坏情况发生在**有序插入**时，树退化为链表。
- 如果你发现自己的 BST 在有序数据上表现很差，这是正常的——这正是后续学习 AVL 树、红黑树的动机。

### 关于测试

- 用中序遍历验证 BST 性质是最简单有效的方法。
- 测试删除操作时，务必覆盖三种情况（叶子、单子节点、双子节点）。
- 测试边界情况：空树操作、只有一个节点的树、重复 key 插入。
